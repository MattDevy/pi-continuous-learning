## Codebase Patterns
- findSkillPromotionCandidates is in instinct-skill-promotions.ts (not generators); takes (instincts, mergeClusters: Instinct[][], shadowIds) - uses Instinct[][] not MergeSuggestion[] to avoid circular deps.
- Cluster members tracked in clusterMemberIds Set inside findSkillPromotionCandidates; singles pass skips those IDs to prevent double-reporting.
- instinct-evolve-generators.ts extracts mergeClusters = mergeSuggestions.map(s => s.instincts) then shadowIds = new Set(skillShadows.map(s => s.instinct.id)) before calling findSkillPromotionCandidates.
- tokenizeText lives in instinct-text-utils.ts (not generators); import from there for any new dedup/overlap code. generators.ts re-exports it for backward compat.
- Skill shadow detection is in instinct-skill-shadows.ts; generators.ts re-exports findSkillShadows/SkillShadowSuggestion/SKILL_SHADOW_TOKEN_THRESHOLD.
- Token overlap tests: tokenizeText does NOT stem (debugging != debug, errors != error, logs != log). Write tests using exact token matches.
- loadSkills() from @mariozechner/pi-coding-agent discovers all configured skill dirs; returns { skills: Skill[] }. Skill.name and Skill.description are the fields needed for InstalledSkill.
- installedSkills state var in index.ts is populated at session_start; initialized to [] so EVOLVE_CMD handler always receives an array (never undefined from closure).

## Codebase Patterns
- findAgentsMdAdditions() takes a `scope` param ('project'|'global') and applies scope-specific thresholds (0.75/0.8); call it twice from generateEvolveSuggestions to get both sets.
- overlapIds (Set<string>) is computed inside generateEvolveSuggestions from overlapSuggestions before calling findAgentsMdAdditions; prevents adding flagged instincts to addition suggestions.
- proposedBullet is generated by buildProposedBullet() - strips leading "when " from trigger, strips trailing punctuation from action, formats as "When {trigger}, {action}."
- findAgentsMdOverlaps() uses a "coverage" metric (matching tokens / instinct tokens) rather than Jaccard; this is intentional - we care how much of the instinct is covered by AGENTS.md, not symmetric overlap.
- AgentsMdOverlapSuggestion is appended last in generateEvolveSuggestions() result array; formatter renders it as '## Duplicates AGENTS.md' section.
- findExcerpt() scans AGENTS.md line-by-line for the first line containing any overlapping token; falls back to first 100 chars of the full text.
- generateEvolveSuggestions() combines agentsMdProject + agentsMdGlobal with a newline join before calling findAgentsMdOverlaps(); null/undefined params are filtered out before join.

- Add reusable, project-wide implementation patterns here.
- agents-md.ts is the canonical entry point for reading AGENTS.md files; always use readAgentsMd() - never read directly - to get null-safe behavior.
- buildAnalyzerUserPrompt accepts an optional AnalyzerUserPromptOptions object; callers that don't need AGENTS.md/skills can omit it entirely (defaults to empty).
- System prompt sections are built by private helper functions (buildXSection()) and assembled in buildAnalyzerSystemPrompt(); add new sections by adding a helper and appending to the sections array.
